WHAT'S A GOOD CSV LOOK LIKE?

Anything with a header row and one comic per row. Common columns Longbox can map to:

  • Series & Title — used to search across sources.
  • Issue Number, Series Year, Publisher, Type (TPB / HC / OMNIBUS / SINGLE_ISSUE / OGN).
  • ISBN-13 or UPC if you have them — best chance of an exact match.
  • Fandom — auto-tagged on each comic.
  • Collected Issues, Variant — saved as-is.

Other columns are fine — you'll get a chance to pick which ones map to which field on the next step. Empty rows and section-header rows (cells with only a fandom name) are skipped automatically.
